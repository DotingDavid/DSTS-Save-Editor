[size=4][color=#00BFFF][b]Backup System[/b][/color][/size]

The editor has a multi-layered backup system to protect your save files:

[color=#81C784][b]Automatic Backups.[/b][/color] A backup is created before every save, stored in a [b]backups[/b] folder next to your saves. The editor keeps the 2 most recent backups per slot. If the backup fails for any reason, the save is aborted — your original file is never overwritten without a successful backup.

[color=#81C784][b]File Manager.[/b][/color] The built-in File Manager lets you manage backups per save slot, view timestamps, and restore at any time.

[size=4][color=#00BFFF][b]Important Notes[/b][/color][/size]

[b]Close the game before editing.[/b] The game's autosave will overwrite your edits if it's running. The editor warns you if it detects the game process.

[b]Stay within reasonable values.[/b] The game displays stats up to 9999, and the editor enforces this cap. Setting extreme values on low-level Digimon may cause unexpected behavior in-game.

[b]Level changes are smart.[/b] When you change a Digimon's level, the editor automatically sets the correct EXP for that species' experience curve and adjusts growth stats to match.

[b]What we don't touch.[/b] The editor only modifies verified fields. It does not change quest progress, dialogue history, encounter data, or any unverified byte ranges.

[size=4][color=#00BFFF][b]How It Works[/b][/color][/size]

The game encrypts save files with AES-128-ECB. The editor decrypts the file, parses the binary roster structure using the game's own flags, and presents it through a graphical interface. When you save, it encrypts and writes back.

The roster parser matches the game's exact displayed counts. No heuristic deduplication or guesswork.
